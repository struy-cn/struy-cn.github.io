AI 视频都在浏览器里生成，剪辑回到 Premiere → 下载 → 导入 → 替换 → 重复。一个 YC S24 团队受够了这个循环，自己造了个东西。

Palmier Pro 的实际编辑器界面（本地实测截图）

他们解决了一个很真实的痛点

团队之前一直在帮其他公司做 AI 发布视频。流程是这样的：

在网页端生成 → 下载到本地 → 拖进剪辑软件 → 替换旧片段 → 重新调参数 → 重复 10 遍

同时项目管理也是一团乱，每一镜有十几个版本，”shot_v3_final_really_final.mp4” 式的命名随处可见。上下文还分散在不同 AI 工具里：Claude 写脚本，AI 平台负责生成，互不通气。

所以他们决定从头写一个剪辑软件。编辑器是唯一的事实来源（single source of truth），你的 Agent 通过 MCP 直接操作时间线。

它和 Premiere / CapCut 有什么不一样

Palmier 自己承认：纯剪辑功能还很基础。没有特效、没有转场、没有调色、没有遮罩、没有图形。但这恰恰是它的设计取舍——它不为传统剪辑而生，为 AI 视频的迭代场景而生。

对比一下：

Runway / Artlist Palmier Pro
核心 生成平台 剪辑软件
生成方式 网页端生成→下载 时间线里直接生成
迭代 下载+替换 原地重生成
成本 每次迭代都要下载导入导出 一个项目搞定

第三方素材也可以混进来，跟 AI 生成的片段在同一条时间线上剪辑，最后统一导出 MP4 或 XML（可对接 Premiere / DaVinci）。

MCP 接入：Agent 操作剪辑时间线

这是 Palmier Pro 最与众不同的地方。打开编辑器后，它在本地暴露一个 MCP 服务端：

然后你的 Agent 就可以接入：

Claude Code

Cursor — Help → MCP Instructions → Install in Cursor 一键安装

Claude Desktop — 自带 mcpb 打包，一键安装 Desktop Extension

接入之后，Agent 能做什么？读你的项目结构、看有哪些媒体片段、在时间线上添加/替换片段、调用内嵌的 AI 生成模型。你在聊天窗里跟 Agent 说”第三镜换成 16:9 的无人机俯拍”，它直接在时间线上执行。

内嵌的 AI 模型包括 Seedance、Kling、Nano Banana Pro 等当前热门的文生视频模型。

MCP 接口实测通过：

  • 文生视频：Seedance 2（含 Fast）、Kling O3 / V3、Veo 3.1（含 Fast / Lite）、Grok Imagine Video、Hailuo 2.3 Pro 等
  • 文生图：Nano Banana Pro/2、GPT Image 2、Grok Imagine、Krea 2 Large 等
  • 音频：ElevenLabs TTS、Gemini TTS、Minimax Music 等
  • 增强：ByteDance / SeedVR / Topaz 超分

开源 + 免费策略

GPLv3 开源，编辑器本体完全免费、无需登录。MCP 服务端和 Agent 对话也是开源的。闭源部分只有生成式 AI 的处理逻辑，这部分需要登录和订阅。

  • GitHub: github.com/palmier-io/palmier-pro
  • 官网: palmier.io

说人话版本

如果你只是在剪 Vlog 或者做宣传片，Palmier Pro 现在的功能还不够完备。

但如果你在大量迭代 AI 视频，一个镜头生成几十版、反复对比效果——它的工作流价值就很明显了。生成-剪辑-迭代在一个软件里完成，Agent 帮你操作时间线。这是一个验证「AI + 剪辑软件应该长什么样」的实验品，而且实验方向很有意思。
