建议栏目：AI 产品观察 / 效率工具选型 / 腾讯 AI Agent 矩阵

如果最近你关注腾讯的 AI 产品，很容易被三个名字绕晕：QClaw、WorkBuddy、Marvis。它们都打着 AI Agent、办公效率、个人助手的标签，也都在强调能帮用户“干活”。但真正放到使用场景里看，它们并不是简单的替代关系，而更像腾讯在个人生产力场景里铺开的三条路线。

本文先给一个最短结论：QClaw 像“电脑遥控器”，WorkBuddy 像“AI 虚拟团队”，Marvis 像“系统级个人管家”。如果你只是想在手机上给电脑发一句话，让电脑替你处理文件、网页或邮件，QClaw 更贴近需求；如果你想让 AI 帮你产出调研报告、PPT、数据分析或内容方案，WorkBuddy 更像一个能交付成果的工作台；如果你想让 AI 深入本地文件、系统设置、跨端接管和隐私场景，Marvis 更像住进操作系统里的私人助手。

一、先把三个产品放回各自的位置

QClaw 的官方描述非常直白，它强调“随时随地，微信一下，QClaw 帮你高效干活”，核心卖点包括通过微信直接对话远程操控电脑、支持丰富 Skills 生态、具备持续记忆，并且可以本地部署到 macOS 与 Windows 设备上。1 腾讯全球新闻稿进一步把 QClaw 定位为降低 AI Agent 部署门槛的产品：用户下载安装、注册并扫码后，大约三分钟就能开始使用；它基于开源 OpenClaw 框架，可接入多个模型，也支持用户通过 API key 配置自定义模型。2

腾讯对 QClaw 的叙事重点并不是“替你写一份报告”，而是“让你的电脑能被聊天应用唤醒并执行任务”。这决定了它的第一身份是本地电脑 Agent 与远程控制入口。

图注：QClaw 官网首屏突出展示了“微信直接对话，远程操控”“丰富 Skills 生态”“持续记忆”和“本地部署”等关键词，这也解释了它为什么更像一个手机聊天框里的电脑遥控器。

WorkBuddy 的气质则完全不同。它来自腾讯云代码助手 CodeBuddy 的办公 Agent 产品线，官网定位为“AI 专家团 全场景办公”，强调“一人指挥，全行业专家执行，从策略到交付一站搞定”。官网列出的能力包括多专家、多模型协同，桌面端、主流 IM 与小程序接入，100+ 领域专家，任务自主规划，以及 MCP 生态与自定义 Skills 扩展。3 腾讯云官方新闻稿也将 WorkBuddy 称为 agentic AI workspace，强调它可以把一句指令转化为完整可用的交付物，并支持多个任务和 agent 并行运行。4

WorkBuddy 的关键词不是“控制电脑”，而是“交付工作成果”。它更接近一个由策略、运营、数据、设计、开发等虚拟角色组成的AI 办公团队。

图注：WorkBuddy 官网将产品主视觉定义为“AI 专家团 全场景办公”，并通过策略、运营、产品经理、数据、开发等机器人角色，强化“多人协作式交付”的产品心智。

Marvis 的页面表达又回到个人设备本身。Marvis 官网写着“马维斯 为你 24 小时随时在线”，并显著强调“本地大模型，本地模式文件 0 上传”。它支持 Windows、macOS 与 Android，并展示了手机操控电脑、文件智能整理搜索、一句话完成电脑设置、文件深度理解与生成等能力。5 这意味着 Marvis 的重心不只是远程发指令，也不只是产出办公文档，而是更深地嵌入个人设备、文件系统和跨端控制。

Marvis 更像一个面向个人电脑和移动端的系统级 AI 中枢。它关心的是你的本地文件在哪里、电脑如何设置、任务执行画面能否被手机接管，以及隐私模式下文件是否需要上传。

图注：Marvis 官网首屏强调 Windows、macOS、Android 多端下载，并在“更懂你的AI助手”板块突出本地大模型、本地模式文件 0 上传等特征，因此更适合被理解为系统级个人 AI 管家。

二、核心差异：入口不同，任务不同，深度也不同

三者都可以被归入“腾讯 AI Agent 产品矩阵”，但如果只用“AI 助手”概括，就会遮蔽最关键的差异。真正决定用户该选谁的，是三件事：从哪里进入、让它做什么、它能深入到系统的哪一层。

对比维度 QClaw WorkBuddy Marvis
一句话定位 手机聊天框里的电脑遥控器 能交付成果的 AI 专家团 系统级个人 AI 管家
主要入口 微信/IM 与桌面端 桌面端、主流 IM、小程序 电脑端与手机跨端控制
核心任务 远程操作电脑、处理文件网页、邮件提醒、自动化任务 调研、PPT、数据分析、内容生产、业务洞察 文件搜索整理、系统设置、格式转换、知识库、本地理解
技术气质 本地 Agent、Skills、记忆、安全网关 多 Agent 并行、多专家/多模型、MCP 与 Skills 扩展 本地大模型、端侧模式、系统权限、文件与图片理解
适合人群 想随时远程让自己电脑干活的个人用户 创业者、职场人、运营/市场/数据等结果导向用户 重视本地文件、隐私、设备管理和生活场景的个人用户

把它们放在同一张表里，差异会更明显。QClaw 的最小闭环是“我在手机上发一句话，我的电脑替我执行”。它将微信或其他 IM 变成远程控制入口，适合处理“我现在不在电脑前，但电脑上有我需要的文件、网页、软件或工作环境”这类问题。1 2

WorkBuddy 的最小闭环是“我交代一个业务目标，它组织多个专家角色完成交付”。例如做竞品调研、生成营销方案、分析业务数据、产出汇报材料。它的价值不在于是否接管你的本地电脑，而在于把复杂办公任务拆成多个子任务，并以“结果件”的方式返回。3 4

Marvis 的最小闭环是“我把个人设备和本地文件交给一个更懂系统的助手管理”。它强调本地模式、文件 0 上传、手机随时操控电脑、实时查看执行画面与接管任务，这说明它在个人设备层面的存在感更强。5

三、用三个典型场景来理解

假设你在外面开会，突然需要电脑上的一个文件，并希望顺手整理网页资料。这个场景下，QClaw 的逻辑最顺，因为它就是围绕“手机远程唤醒电脑”设计的。你把需求发给聊天框，电脑端 Agent 在本机环境里执行，文件、浏览器、邮件等本地资源都可以成为它的操作对象。1

假设你要在两小时内做一份行业分析 PPT，不仅需要检索资料，还要提炼观点、生成结构、可能还要做图表和汇报口径。这个场景下，WorkBuddy 更合适，因为它的产品叙事是“AI 专家团”和“从策略到交付一站搞定”。它并不是把电脑当成被控对象，而是把任务当成可拆解、可并行、可交付的工作流。3 4

假设你的问题是“帮我找到最近下载过的合同，提取重点，顺便把电脑某项设置改好，再把整理结果同步给手机查看”。这个场景下，Marvis 的优势更明显，因为它强调对本地文件、图片文字、系统设置和跨端接管的理解。尤其是本地模式文件 0 上传的表述，说明它更关注个人数据、隐私和设备层任务。5

四、一个更直观的选型方法

如果你只想快速判断，可以问自己三个问题。第一，你是否经常“不在电脑前，但希望电脑继续替你干活”？如果是，优先看 QClaw。第二，你是否更关心“最后能不能给我一份完整报告、PPT、分析或方案”？如果是，优先看 WorkBuddy。第三，你是否更关心“本地文件、系统设置、跨端接管与隐私模式”？如果是，优先看 Marvis。

当然，对于重度用户来说，这三者未必是三选一。一个更合理的组合是：QClaw 管远程唤起，WorkBuddy 管成果交付，Marvis 管本地系统与知识库。换句话说，QClaw 解决“怎么从手机触达电脑”，WorkBuddy 解决“怎么把任务变成工作成果”，Marvis 解决“怎么让 AI 更懂我的设备与本地资料”。

五、为什么腾讯会同时做三个？

从产品矩阵看，腾讯并不是只押注一种 Agent 形态，而是在同时试探三个入口。第一是通讯入口，也就是 QClaw 代表的 IM 控制电脑：用户已经习惯在微信、QQ、企业微信、飞书、钉钉等工具里发指令，如果 Agent 能从这些入口进入，学习成本最低。第二是办公交付入口，也就是 WorkBuddy 代表的 agentic workspace：用户不只是要聊天，还要报告、方案、图表、PPT 和业务洞察。第三是设备系统入口，也就是 Marvis 代表的本地 AI 管家：当模型能够理解文件、屏幕、系统设置和跨端任务时，Agent 会更接近个人操作系统的一部分。

这也解释了为什么三者看上去相似，却不会完全重叠。QClaw 的中心是“电脑执行”，WorkBuddy 的中心是“工作成果”，Marvis 的中心是“个人设备与本地上下文”。它们都在回答“AI 怎么替人干活”，但对“活”的定义不同。

六、最后的判断

如果要用一句话收束：QClaw 是入口型 Agent，WorkBuddy 是交付型 Agent，Marvis 是系统型 Agent。QClaw 让你更容易远程驱动电脑，WorkBuddy 让你更容易拿到完整办公成果，Marvis 让 AI 更深入个人设备、文件和跨端场景。

对普通用户来说，最务实的选择不是追最新名字，而是先判断自己的主要痛点。如果痛点是“我不在电脑前”，选 QClaw；如果痛点是“我需要一份能用的工作成果”，选 WorkBuddy；如果痛点是“我的本地资料和设备需要一个长期管家”，选 Marvis。

未来真正有意思的地方，可能不是三者谁取代谁，而是这些能力会不会在腾讯生态里逐渐打通。当通讯入口、办公交付和系统级本地理解连在一起时，AI Agent 才真正从“会聊天的工具”，走向“能持续替你处理事务的数字劳动力”。
