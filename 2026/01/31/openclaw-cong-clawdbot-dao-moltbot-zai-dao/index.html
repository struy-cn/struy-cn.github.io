2026 年开年，硅谷最火的开源项目大概就是这个“龙虾系”AI 代理了——它先叫 Clawdbot（因为致敬 Claude 的小怪兽），72 小时内 GitHub 星标狂飙 6 万+，直接带火 Mac Mini 销量（无数人冲去买一台当 24/7 专用服务器），Discord 社区破万，国内外各路 AI 博主纷纷转发蹭流量。

结果 Anthropic 发来一封“礼貌的”邮件，指出名字太像 Claude，项目被迫改名 Moltbot（龙虾蜕壳的意思，挺诗意）。可改名窗口期被 crypto 骗子抢注账号、搞假币，混乱中又紧急再改一次——现在正式叫 OpenClaw，开源 + 龙虾爪，算是尘埃落定。

OpenClaw 是什么：开源智能体网关

OpenClaw 的本质是一个智能体网关（AI Agent Gateway）。它由退休后再复出的软件工程师 Peter Steinberger 开发，并于 2025 年底在其个人工作流文章中首次亮相，2026 年初在硅谷开发者社区迅速传播。

与 ChatGPT、Claude 等云端 AI 助手不同，OpenClaw 的核心设计理念是本地执行。它通常运行在一台 Mac mini（或任何本地计算机、云服务器）上，充当两个关键角色：

AI 智能体：调用 Claude Opus、Gemini 等顶级大模型的 API，进行思考与决策。
网关（Gateway）：连接 WhatsApp、Telegram、iMessage 等聊天应用，接收你的指令，并将 AI 的回复返回给你。

这意味着，你不再需要打开专门的网页或 App。
只需像给朋友发消息一样，比如在 WhatsApp 里输入“帮我整理今天的邮件”，OpenClaw 就会在后台操作你的电脑，完成实际任务。

但争议也没停过

有人惊呼“这是个人 AI 助手的未来”

有人警告“它能访问你全部数字生活，隐私和安全风险极高”

甚至有研究发现大量实例暴露了控制面板，随时可能泄露凭证。
创始人 Peter Steinberger 自己都反复提醒：别乱买 Mac Mini，别在 VPS 公开暴露端口。

尽管如此，OpenClaw 依然在病毒式传播。它不是云端聊天机器人，而是真正跑在你本地、能“做事”的代理：通过 WhatsApp、Telegram、iMessage 等聊天发指令，它就能帮你清空收件箱、订餐、管日程、写代码、订机票……像一个永不睡觉的私人助理。

一个真实用例，用他来干啥

我最近把 ActivityWatch 和 OpenClaw 结合起来玩，感觉打开了新世界：前者负责“觉察”，后者负责“行动 + 优化”。

ActivityWatch 是一个完全本地的开源时间追踪工具。它默默记录你用了哪些 App、开了哪些窗口、浏览了哪些网站，所有数据都存在你电脑上，不会上传云端。

安装后，你能看到非常直观的日报、周报：

  • 顶级应用占比
  • 具体窗口标题时间分布
  • 浏览器域名使用时长

比如我一看才发现：一天 4 小时在浏览器里“研究”，但其实 70% 是刷 X 和无关 tab。这就是典型的“不知不觉浪费”。

官网：https://activitywatch.net/

强烈建议每个人先装一个，跑一周再看数据

OpenClaw + ActivityWatch：觉察 → 复盘 → 自动化闭环

最爽的玩法是把 ActivityWatch 的日志喂给 OpenClaw，让它帮你做“每日行为复盘”。

你可以直接在 WhatsApp / Telegram / 飞书 里问：

  • “分析我昨天的 ActivityWatch 日志，我到底在干嘛？”
  • “我刷 X 的时间是不是太多了？给我个优化建议。”
  • “基于我这周的窗口使用模式，帮我写个 Pomodoro + 自动屏蔽脚本。”

OpenClaw 会读日志、总结问题、甚至直接生成可执行的工作流（比如自动在特定时间锁某些网站、提醒切换任务）。

更进一步：如果你反复做某些重复操作（比如每天汇总邮件、整理 Notion、跑固定报告），OpenClaw 可以观察 ActivityWatch 的模式后，主动帮你创建自动化技能（skill），以后一键或定时就搞定。

工作流大概是：

  1. 觉察：ActivityWatch 全天无感记录
  2. 复盘：每天/每周喂给 OpenClaw 做总结 + 提优化点
  3. 行动：OpenClaw 直接执行改进（自动化 + 提醒 + 干预）

小结：拥抱，但别盲目

OpenClaw 代表了“本地 + 能做事”的下一代个人 AI 方向，它爆火、改名、带货 Mac Mini、引发安全争议，其实都说明一个点：大家真的很渴望一个“懂你、帮你干活、又不把数据全交给大厂”的助手。

但用之前请注意：

  • 跑在本地，不熟使用虚拟机，别暴露端口到公网
  • 敏感账号授权要谨慎
  • 先小范围测试，别一下全盘托付
  • 谨慎点，别让他摧毁了你的电脑😂
  • 对硬件要求不高，但需要大模型 API KEY 以及各种配置，需要折腾。
  • 不需要专门为了使用单独买个机器做这件事，没必要
  • 没有配置技能和使用场景，这和普通聊天助手没有区别，主要是交互方式不同而已

越是便捷，隐私性差。AI 时代好像什么都暴露给 AI 了，但没有上下文的 AI 的确没法实际工作。
